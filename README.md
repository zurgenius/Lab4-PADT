# Lab 4: LazySequence, Stream и онлайн-статистика

Проект реализует четвертую лабораторную работу поверх готового ядра ЛР2. Новая функциональность ординальная арифметика, ленивые последовательности, генераторы, потоки и задача варианта 4 "Сбор онлайн-статистики".


## Структура проекта

- `include/` - публичные заголовки.
- `include/detail/` - шаблонные реализации базового ядра ЛР2.
- `include/lazy/` - ленивые последовательности, кэш и генераторы.
- `include/streams/` - потоковые интерфейсы и реализации потоков.
- `include/tasks/` - прикладная задача онлайн-статистики.
- `src/` - нетемплейтные реализации, ординальная арифметика и консольное меню.
- `tests/` - модульные тесты.

## Команды

- `make program` - сконфигурировать CMake, собрать программу и обновить локальный `./program`.
- `make tests` - сконфигурировать CMake, собрать тесты и запустить `ctest`.
- `make lint` - собрать программу и тесты через отдельный CMake build с `-Werror`.
- `make format` - отформатировать код через `clang-format`.
- `cmake -S . -B build` - ручная конфигурация проекта.
- `cmake --build build` - ручная сборка целей.
- `ctest --test-dir build --output-on-failure` - ручной запуск тестов.

При первой CMake-сборке GoogleTest подтягивается автоматически через `FetchContent`.

## Архитектурная идея
Добавлены

- `OrdinalIndex` и `OrdinalLength` для индексов и длин с бесконечными блоками.
- `Generator<T>` как абстракция вычисления элементов.
- `LazySequence<T>` как последовательность, которая не строит результат операции сразу, а хранит генератор и материализует только запрошенный префикс.
- `Cache<T>` как ограниченное окно истории последних значений, по умолчанию 50 элементов.
- `Stream`-слой для последовательного чтения/записи из разных источников.
- `tasks/event_statistics.h` для обработки протокола событий и подсчета статистики онлайн.

Ленивые операции (`append`, `prepend`, `insert`, `concat`, `interleave`, `map`, `where`, `take`) создают новую `LazySequence<T>` с конкретным генератором. Исходные последовательности не материализуются целиком. Значение вычисляется только при `get(...)`.

### LazySequence

Файлы:

- `include/lazy/lazy_sequence.h`
- `include/lazy/detail/lazy_sequence.tpp`

`LazySequence<T>` наследуется от `Sequence<T>`, но часть методов базового интерфейса намеренно запрещена через `std::logic_error`, потому что они архитектурно некорректны для потенциально бесконечной последовательности. 
Операции работают через генераторы. Также реализован кольцевой кэш.


### Streams

Файлы:

- `include/streams/base.h`
- `include/streams/sequence.h`
- `include/streams/lazy.h`
- `include/streams/file.h`
- `include/streams/live_file.h`
- `include/streams/stream.h`
- `include/streams/detail/*.tpp`

Реализован слой потокового чтения и записи:

- `ReadOnlyStream<T>` - базовый интерфейс чтения.
- `WriteOnlyStream<T>` - базовый интерфейс записи.
- `SequenceReadOnlyStream<T>` - читает конечную `Sequence<T>` слева направо.
- `SequenceWriteOnlyStream<T>` - пишет в `Sequence<T>` через `append`.
- `LazyReadOnlyStream<T>` - читает `LazySequence<T>` и материализует элементы по мере чтения.
- `FileLineReadOnlyStream` - читает строки из обычного файла.
- `FileLineWriteOnlyStream` - пишет строки в файл.
- `LiveFileLineReadOnlyStream` - читает файл, который может дописываться другим приложением.

Обычный файловый поток считает EOF концом файла. Live-поток при временном отсутствии полной новой строки возвращает `None` из `try_read()`, ждет новые данные и завершает чтение только по правилу стабильного EOF.

### Онлайн-статистика событий

Файлы:

- `include/tasks/event_statistics.h`
- `include/tasks/detail/event_statistics.tpp`

Реализован вариант 4 "Сбор онлайн-статистики".

Типы:

- `EventType` - `Start`, `End`, `Measure`, `Error`, `Unknown`.
- `Event<T>` - событие с типом, числовым значением и сообщением.
- `EventParser<T>` - парсер строк протокола.
- `EventReadOnlyStream<T>` - адаптер потока строк в поток событий.
- `OnlineEventStatistics<T>` - накопитель статистики.
- `ProtocolStatisticsTask<T>` - обработка всего протокола.
- `EventBatchProcessingTask<T>` - обработка протокола батчами.

Формат протокола:

- `START ...`
- `END ...`
- `MEASURE <number>`
- `ERROR ...`
- остальные строки считаются `Unknown`.

Статистика считает:

- общее количество событий;
- количество `Start`, `End`, `Measure`, `Error`, `Unknown`;
- минимум, максимум, сумму, среднее и дисперсию для `Measure`.
